# 制作背景

* バイクを買ったはいいが,一緒にツーリングにいく仲間がいない人が、遠くの人とも気軽に行けるようにツーリングプランを企画して実際にユーザー同士が繋がれるアプリがあれば、バイクを所持していて身近にバイクに興味のある人がいないユーザーに役に立つのではないかという風に考えた。

# 工夫したポイント

* 最初はcssやscssを自力で打ち込んでいたが、記述量が多くなり複雑になってきたので、bootstrap4を導入。記述量が大幅に減少し、
* コーディングの効率がとても上がった。

* UI/UXを意識し,詳細画面や各ボタンなどわかりやすい色にして,一目でどういうボタンかを判別しやすくした。

* 当初はデプロイするにあたってherokuにデプロイをしていたが、実践を意識してawsにデプロイをした。
  herokuの場合は応答レスポンスが遅かったり、画像が表示されない等の一部制約があるので、より受難性の高い
  awsにデプロイをした。

* 参加したプラン、自分が企画したプランを判別するために上部コンテンツにドロップダウンを用いて、プランを素早く区別することが
  できるようにした。

* プラン一覧表示の際に「参加済みです」の表示をすることによって自分が参加したプランとわかるようにした。

* アプリ説明をする際にカルーセルを用いることによって、コンパクトに表示ができたので閲覧しやすくなった。

* 登録者一覧画面にて,全てのユーザーを表示してどういったユーザーがいるのかをわかるようにした。

# アプリの利用方法

* 各機能について
* ユーザー登録をしていない場合
  上部のユーザー登録、ログイン欄にて新規登録画面及びログイン画面でユーザー登録ができます。
  ツーリングプランを閲覧することはできますが、プランを企画したり、参加することができません。

* ユーザー登録後の場合
  ユーザー登録欄の自身のニックネーム部をクリックすると、自身のプロフィールが閲覧できます。
  また、ツーリングを企画することができます。企画して正常に作成出来た企画は「TOURING PLAN」の下に表示されます。
  加えて、ツーリングプランに参加することができます。

  ツーリングプランの詳細ボタンを押すとツーリングの詳細画面を閲覧することができます。
  また、ツーリング企画者のみツーリング企画を削除することができます。

# 目指した課題解決

* どのような人
  バイクを所持している全てのユーザー

* どのような課題解決
  埼玉県の高校では2019年４月に、バイクに乗らない、バイクを買わない、バイクの免許を取らないという「３ない運動」が廃止され、高校生でも気軽にバイクを購入したり免許を取得できるようになった。しかし、高校生活などでしょっちゅう集会などに顔を出せるわけではないので、このようなサイトがあれば気軽に開いた時間でも自分の空いているスケジュールのツーリングをさがしだせる上に、遠く離れたユーザーとも知り合えてより多くのコミュニケーションを行うことができると考えた。

# 要件定義

* ユーザーを識別し、ユーザ間での理想の仲間を見つけやすくする。-> ユーザー登録機能

* ユーザープロフィールを編集したい。 -> ユーザー編集機能

* ログアウトしたい。 -> ユーザーログアウト機能

* 自分の詳細なプロフィールを閲覧したい。 -> ユーザー詳細画面

* 実際にツーリングを企画し、たくさんのユーザーとツーリングを行いたい。-> ツーリングプラン作成機能

* 企画してあるプランの詳細を詳しく閲覧してみたい。 -> ツーリングプラン詳細機能

* 企画してあるツーリングに参加してみたい。 -> ツーリングプラン参加機能

# 機能説明
* ユーザー新規登録機能,,,上部のユーザー登録欄にて新規登録可能

* ログイン機能,,,上部のユーザー登録欄にてログイン可能

* ログアウト機能,,,上部のユーザー登録欄にてログアウト可能

* ユーザー詳細機能,,,上部のユーザー登録欄にて詳細画面へ移動可能

* ツーリングプラン企画機能,,,上部コンテンツにて企画可能。(ユーザー登録者のみ)

* ツーリングプラン参加機能,,,ユーザー登録しているかつ、自分が企画していないプランの詳細画面にて参加可能
